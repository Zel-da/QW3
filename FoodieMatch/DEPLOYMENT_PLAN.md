## 2. 도입 단계

### Phase 1: 내부 테스트 (1주)
**목표**: 개발팀 + 안전보건팀 핵심 담당자 2~3명

| 항목 | 담당 | 기간 |
|------|------|------|
| 테스트 계정 생성 | 개발팀 | 1일 |
| 주요 기능 시연 | 개발팀 → 안전보건팀 | 1시간 |
| 실제 데이터로 TBM 작성 테스트 | 안전보건팀 | 3일 |
| 버그/개선사항 수집 | 양측 | 지속 |
| 긴급 수정 | 개발팀 | 2일 |

**산출물**:
- 테스트 결과 보고서
- 개선 요청 목록

---

### Phase 2: 파일럿 운영 (2주)
**목표**: 1~2개 팀 선정하여 실제 업무에 적용

**추천 파일럿 팀**:
- BR 생산관리 (TBM 주관팀)
- CR 조립 (인원 많은 팀)

| 항목 | 담당 | 기간 |
|------|------|------|
| 파일럿 팀 선정 | 안전보건팀 | 1일 |
| 팀장 교육 (1:1) | 개발팀 | 2시간/팀 |
| 실제 TBM 작성 운영 | 파일럿 팀 | 2주 |
| 일일 피드백 수집 | 안전보건팀 | 매일 |
| 주간 회의 | 전체 | 주 1회 |

**체크포인트**:
- [ ] TBM 작성 시간 측정 (기존 대비)
- [ ] 사용자 만족도 조사
- [ ] 시스템 안정성 확인

---

### Phase 3: 전사 교육 (1주)
**목표**: 전 팀 대상 사용법 교육

| 대상 | 교육 내용 | 시간 | 방식 |
|------|----------|------|------|
| 팀장/관리자 | 전체 기능 + 결재 | 1시간 | 집합교육 |
| 일반 작업자 | TBM 참여 + 서명 | 30분 | 팀별 교육 |
| 안전보건팀 | 관리자 기능 + 통계 | 2시간 | 1:1 교육 |

**교육 자료**:
- [ ] 사용자 매뉴얼 (PDF)
- [ ] 빠른 시작 가이드 (1페이지)
- [ ] 자주 묻는 질문 (FAQ)

---

### Phase 4: 전사 배포 (2주)
**목표**: 순차적 확대 적용

| 주차 | 대상 | 팀 수 |
|------|------|-------|
| 1주차 | 아산 공장 전체 | ~10팀 |
| 2주차 | 화성 공장 전체 | ~10팀 |

**지원 체계**:
- 헬프데스크 운영 (초기 2주)
- 카카오톡/Teams 문의 채널
- 긴급 이슈 핫라인

## 3. 안전보건팀 협의 안건

### 첫 미팅 (킥오프)
1. **시스템 시연** (30분)
   - 실제 화면으로 주요 기능 데모
   - Q&A

2. **논의 사항**
   - 기존 TBM 양식과 차이점 확인
   - 추가 필요 기능 논의
   - 파일럿 팀 선정
   - 교육 일정 협의

3. **결정 필요 사항**
   - [ ] 파일럿 팀 확정
   - [ ] 테스트 시작일
   - [ ] 안전보건팀 담당자 지정

### 준비할 자료
- 시스템 URL 및 테스트 계정
- 기능 소개 PPT (10장 이내)
- 기존 엑셀 양식 vs 새 시스템 비교표
