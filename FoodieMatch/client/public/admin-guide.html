ADMIN 전용 문서

1. 신규 사용자 승인

승인 절차

  1. 사용자 관리 페이지 접속
    상단 메뉴 "더보기" → "사용자 관리" 클릭
  2. 대기 중인 사용자 필터링
    역할 필터에서 "대기(PENDING)" 선택
  3. 신청 정보 확인
    사용자명, 이름, 이메일, 가입일시 확인. 실제 직원인지 확인 필요
  4. 역할 지정
    "역할 변경" 버튼 클릭 → 역할 선택
    • 팀장 (TEAM_LEADER): TBM 작성, 안전점검, 결재 요청
    • 임원 (EXECUTIVE_LEADER): 팀장 권한 + 결재 승인/반려 (이중 역할)
    • 결재자 (APPROVER): 월별 보고서 승인/반려
    • 관리자 (ADMIN): 모든 시스템 관리 권한
  5. 소속 현장 및 팀 지정
    현장 선택 → 팀 선택
  6. 저장
    "저장" 버튼 클릭 → 승인 완료

  1. 해당 사용자의 "삭제" 버튼 클릭
  2. 확인 대화상자에서 "삭제" 클릭
  3. 삭제된 계정은 복구 불가

2. 사용자 정보 관리

사용자 검색

검색창에서 다음 정보로 사용자를 검색할 수 있습니다:

  • 사용자명 (아이디)
  • 이름
  • 이메일

역할 변경

팀원이 팀장으로 승진한 경우
  1. 사용자 관리에서 해당 사용자 검색
  2. "역할 변경" 버튼 클릭
  3. 역할을 "TEAM_LEADER"로 변경
  4. 필요시 팀도 함께 변경
  5. "저장" 클릭
변경 완료 후 사용자가 다시 로그인하면 새 권한이 적용됩니다.
임원이 팀장 역할도 겸해야 하는 경우
  1. 사용자 관리에서 해당 임원 검색
  2. "역할 변경" 버튼 클릭
  3. 역할을 "EXECUTIVE_LEADER"로 변경
  4. 소속 팀 지정
  5. "저장" 클릭
EXECUTIVE_LEADER는 팀장(TBM 작성, 안전점검, 결재 요청) + 결재자(승인/반려) 권한을 동시에 갖는 이중 역할입니다.

팀 변경

사용자가 다른 팀으로 이동한 경우
  1. 사용자 관리에서 해당 사용자 검색
  2. "팀 변경" 버튼 클릭
  3. 새로운 현장 선택 (필요시)
  4. 새로운 팀 선택
  5. "저장" 클릭
팀 변경 시 이전 팀에서 작성한 TBM, 점검 기록은 유지됩니다. 단, 새 팀의 TBM/점검 데이터에만 접근 가능합니다.

계정 비활성화

퇴사자 계정 처리

퇴사자의 계정은 삭제하지 않고 역할을 PENDING으로 변경하여 비활성화합니다.

  1. 해당 사용자의 "역할 변경" 클릭
  2. 역할을 "PENDING"으로 변경
  3. 저장

* 데이터 보존을 위해 완전 삭제보다 비활성화를 권장합니다.

3. 비밀번호 초기화

사용자가 비밀번호를 분실했을 때 관리자가 임시 비밀번호를 발급할 수 있습니다.

비밀번호 초기화 절차

  1. 사용자 확인
    검색창에서 비밀번호 초기화가 필요한 사용자 검색
  2. 비밀번호 리셋 실행
    해당 사용자 행의 "비밀번호 리셋" 버튼 클릭
  3. 임시 비밀번호 확인
    생성된 임시 비밀번호가 화면에 표시됨
    예시: Temp#a7x9K2m
  4. 사용자에게 전달
    임시 비밀번호를 사용자에게 안전하게 전달 (전화, 대면 등)
  5. 비밀번호 변경 안내
    사용자에게 로그인 후 "프로필" → "비밀번호 변경"에서 새 비밀번호로 변경하도록 안내
임시 비밀번호는 화면에 한 번만 표시됩니다. 창을 닫으면 다시 확인할 수 없으므로 반드시 메모하거나 즉시 사용자에게 전달하세요.

사용자 셀프 서비스

사용자가 직접 비밀번호를 재설정할 수 있습니다:
1. 로그인 페이지에서 "아이디/비밀번호를 잊으셨나요?" 클릭
2. "비밀번호 찾기" 탭에서 이메일 입력
3. 이메일로 재설정 링크 수신
4. 링크 클릭하여 새 비밀번호 설정

비밀번호 정책

항목요구사항
최소 길이8자 이상
필수 포함영문자, 숫자, 특수문자 각 1개 이상
금지 항목사용자명, 이름과 동일한 비밀번호

4. 팀/팀원 관리

팀원 관리 개요

TBM 서명을 위해 시스템 계정이 없는 팀원도 등록할 수 있습니다. 등록된 팀원은 TBM 작성 시 참석자로 선택할 수 있습니다.

팀원 추가

  1. 팀원 관리 페이지 접속
    더보기 → 팀원 관리
  2. 팀 선택
    좌측 현장/팀 목록에서 팀원을 추가할 팀 선택
  3. "팀원 추가" 클릭
    우측 상단의 "팀원 추가" 버튼 클릭
  4. 팀원 정보 입력
    아래 필드 입력 후 저장
팀원 추가 필드
이름*필수 팀원의 실명 (한글 2~10자)
직급선택 예: 사원, 대리, 과장, 차장, 부장
연락처선택 휴대전화 번호 (010-0000-0000 형식)

팀장 지정

팀의 팀장을 변경해야 할 때
  1. 팀원 관리에서 해당 팀 선택
  2. 팀장으로 지정할 팀원 찾기
  3. 해당 팀원의 "팀장 지정" 버튼 클릭
  4. 확인 대화상자에서 "확인" 클릭

* 팀장은 TBM 작성 및 월별 보고서 결재 요청 권한을 갖습니다.

결재자 지정

팀의 결재자(임원)를 지정할 때
  1. 팀원 관리에서 해당 팀 선택
  2. "결재자 설정" 버튼 클릭
  3. APPROVER, EXECUTIVE_LEADER 또는 ADMIN 역할을 가진 사용자 중 선택
  4. 저장
결재자는 해당 팀의 월별 보고서를 승인/반려할 수 있습니다. 한 명의 결재자가 여러 팀을 담당할 수 있습니다. EXECUTIVE_LEADER는 자신이 팀장인 팀의 결재 요청도 하면서 다른 팀의 결재 승인도 할 수 있습니다.

팀원 삭제

팀원을 삭제하면 해당 팀원의 TBM 서명 기록은 유지되지만, 이후 TBM 작성 시 참석자로 선택할 수 없습니다. 퇴사자는 삭제보다 "비활성" 상태로 변경하는 것을 권장합니다.

평가 문제 관리

교육에 평가 문제 추가
  1. 교육 관리에서 해당 과정 선택
  2. "문제 관리" 탭 클릭

교육을 삭제하면 해당 교육의 모든 이수 기록도 함께 삭제됩니다. 삭제 대신 "비공개" 상태로 변경하는 것을 권장합니다.

6. 교육 이수 모니터링

교육 현황 확인

팀별, 사용자별 교육 이수 현황을 실시간으로 확인할 수 있습니다.

  1. 교육 현황 페이지 접속
    더보기 → 교육 현황
  2. 필터 설정
    현장, 팀, 교육 과정, 기간 등으로 필터링
  3. 현황 확인
    이수율, 미이수자, 진행 중인 사용자 확인

교육 현황 항목 설명

항목설명
이수 완료영상 시청 100% + 평가 합격
진행 중영상 시청 시작했으나 미완료
미시작아직 교육을 시작하지 않음
평가 불합격영상은 완료했으나 평가 80% 미만

미이수자 알림

교육 기한 임박 알림 발송
  1. 교육 현황에서 미이수자 목록 확인
  2. "알림 발송" 버튼 클릭
  3. 대상자 선택 (전체 또는 개별)
  4. "발송" 클릭
  5. 대상자에게 이메일 알림 발송됨

교육 이수 기록 엑셀 다운로드

  1. 교육 현황 페이지에서 필터 설정
  2. "엑셀 다운로드" 버튼 클릭
  3. 다운로드된 파일에서 상세 기록 확인

다운로드 파일 포함 내용: 사용자명, 이름, 팀, 교육명, 진행률, 평가 점수, 이수일

7. TBM 체크리스트 관리

TBM(작업전 안전점검 회의) 일일점검에서 사용하는 체크리스트 항목을 팀별로 관리합니다.

  4. "항목 추가" 클릭
  5. 점검 항목 내용 입력

8. 안전점검 템플릿 관리

월별 안전점검에서 사용하는 장비 및 점검 항목을 관리합니다.

팀별 장비 등록

데이터베이스 관리 (Neon)

모든 데이터가 저장되는 곳입니다. 직접 데이터 조회, 수정이 가능하지만 신중하게 사용해야 합니다.

데이터 조회 도구 사용

  1. Neon 관리 화면 로그인
  2. 프로젝트 선택
  3. "SQL Editor(데이터 조회)" 탭 클릭
  4. 조회 명령어 입력 및 실행

자주 사용하는 조회 명령어

-- 전체 사용자 수 확인
SELECT COUNT(*) FROM "User";

-- 역할별 사용자 수 확인
SELECT role, COUNT(*) FROM "User" GROUP BY role;

-- 최근 7일 TBM(작업전 안전점검) 작성 현황
SELECT "reportDate", COUNT(*) as count
FROM "TBMReport"
WHERE "reportDate" >= CURRENT_DATE - INTERVAL '7 days'
GROUP BY "reportDate" ORDER BY "reportDate";

-- 교육 이수 현황 확인
SELECT c.title,
       COUNT(CASE WHEN p.completed THEN 1 END) as completed,
       COUNT(p.id) as total
FROM "Course" c
LEFT JOIN "UserProgress" p ON c.id = p."courseId"
GROUP BY c.id, c.title;

저장 용량 확인

  1. Neon 관리 화면에서 "Usage(사용량)" 탭 클릭
  2. 저장 용량 사용량 확인
  3. 처리 시간 사용량 확인
무료 요금제 제한: 저장 용량 512MB, 월 100시간 처리 시간
한도 초과 시 서비스가 중단될 수 있으니 월 1회 이상 확인하세요.

15. 문제 해결

로그인 문제

사용자가 로그인할 수 없는 경우
  • 계정 상태 확인: "승인 대기" 상태인지 확인 (사용자 관리 메뉴)
  • 비밀번호 확인: 필요시 비밀번호 초기화
  • 브라우저 임시 저장 데이터 삭제 후 재시도 안내
  • 브라우저 쿠키 허용 설정 확인

작업전 안전점검(TBM) 문제

TBM이 저장되지 않는 경우
  • 인터넷 연결 상태 확인
  • 필수 입력 항목 확인 (날짜, 참석자, 체크리스트)
  • 서명 입력 여부 확인
  • 같은 날짜에 이미 작성된 TBM이 있는지 확인
TBM이 목록에 보이지 않는 경우
  • 올바른 팀으로 작성되었는지 확인
  • 날짜 필터 범위 확인
  • 저장 버튼을 눌렀는지 확인

교육 관련 문제

교육 진행률이 저장되지 않는 경우
  • 인터넷 연결 상태 확인
  • 영상이 실제로 재생 중인지 확인 (일시정지 상태 아님)
  • 5초 이상 재생 후 자동 저장됨
  • 권장 브라우저 사용 (Chrome, Edge)
교육 영상이 재생되지 않는 경우
  • 유튜브 주소가 올바른지 확인
  • 회사 방화벽에서 유튜브 차단 여부 확인
  • 직접 업로드 영상의 경우 파일 손상 여부 확인

이메일 관련 문제

이메일이 발송되지 않는 경우
  • 이메일 서버 설정 확인 (Render 관리 화면의 서버 설정)
  • 이메일 설정 페이지에서 테스트 메일 발송
  • 받는 사람 이메일 주소가 올바른지 확인
  • 스팸 메일함 확인 안내

시스템 성능 문제

서비스가 느리거나 응답하지 않는 경우
  • Render 관리 화면에서 서비스 상태 확인
  • 기록(Logs) 탭에서 오류 내용 확인
  • 필요시 서비스 재시작 (수동 배포)
  • Neon 관리 화면에서 데이터베이스 연결 상태 확인
데이터베이스 연결 오류
  • Neon 관리 화면에서 데이터베이스 상태 확인
  • 사용량(Usage)에서 한도 초과 여부 확인
